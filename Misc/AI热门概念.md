## 1. Agent（智能体）

**一句话定义：** 能够自主规划、调用工具、完成复杂任务的 AI 系统。

**核心特征：**
- **自主性**：无需人类逐步指令，可自行制定行动计划
- **工具使用**：可调用搜索、代码执行、API、文件系统等外部工具
- **循环推理**：通过"思考 → 行动 → 观察"循环持续迭代
- **记忆管理**：短期上下文记忆 + 长期知识存储

**典型应用：** 自动化研究助手、代码生成与调试 Agent、客服自动化、数据分析 Agent

**代表框架：** LangGraph、AutoGen、CrewAI、Dify

## 2. MCP（Model Context Protocol）

**一句话定义：** 由 Anthropic 提出的开放协议，让 AI 模型能够标准化地连接外部工具和数据源。

**核心价值：**
- **标准化接口**：统一 AI 与外部服务的交互方式，类似 AI 世界的 USB-C
- **即插即用**：开发者只需实现一次 MCP Server，所有兼容模型均可调用
- **安全可控**：权限明确，用户对数据访问保持控制

**架构组成：**
- **MCP Host**：运行 AI 模型的宿主环境（如 Claude Desktop、IDE）
- **MCP Client**：连接并管理 MCP Server 的客户端
- **MCP Server**：提供工具/资源的服务端（如 GitHub MCP、Slack MCP）

**官方资源：** https://modelcontextprotocol.io

## 3. A2A（Agent to Agent）

**一句话定义：** 谷歌提出的开放协议，让不同 AI 智能体之间能够互相通信与协作。

**核心特征：**
- **跨平台协作**：不同厂商、不同框架的 Agent 可以互相调用
- **任务委托**：一个 Agent 可将子任务分配给专属 Agent 完成
- **能力声明**：Agent 通过"Agent Card"公开自身能力和接口
- **与 MCP 互补**：MCP 解决"模型连工具"，A2A 解决"Agent 连 Agent"

**应用场景：** 多智能体协作系统、企业级 AI 工作流自动化、跨系统数据处理

**发布方：** Google（2025年4月发布）

## 4. Claude Code

**一句话定义：** Anthropic 推出的 AI 编程助手，以命令行工具形式深度集成到开发工作流。

**核心能力：**
- **全代码库理解**：可读取并理解整个项目结构，而非仅单个文件
- **自主执行**：可直接运行终端命令、修改文件、执行测试
- **多步骤任务**：能完成"新增功能 → 写测试 → 修复 Bug → 提交代码"的完整链路
- **MCP 集成**：支持通过 MCP 协议扩展工具能力

**使用方式：** 命令行 `claude` 命令，在项目目录中直接交互

**适用场景：** 重构大型代码库、自动化 Code Review、生成技术文档、处理复杂工程任务

## 5. RAG（检索增强生成）

**一句话定义：** 在生成回答前，先从外部知识库检索相关内容，再交给模型生成，从而突破模型知识截止日期的限制。

**工作流程：**
```
用户提问 → 向量检索（语义搜索）→ 召回相关文档 → 拼入上下文 → 模型生成答案
```

**核心组件：**
- **向量数据库**：存储文档的语义向量（如 Pinecone、Weaviate、Chroma）
- **Embedding 模型**：将文本转为向量表示
- **重排序（Rerank）**：对召回结果进行精排，提升准确性

**优势：** 知识可更新、来源可溯源、幻觉率更低

**进化方向：** GraphRAG（图谱增强）、HyDE（假设文档嵌入）、Agentic RAG

## 6. Function Calling（函数调用）

**一句话定义：** 模型识别用户意图后，输出结构化的函数调用指令，由外部系统执行并将结果返回给模型。

**工作原理：**
1. 开发者预先定义可用函数及其参数（JSON Schema）
2. 用户提问时，模型判断是否需要调用函数
3. 模型输出结构化的调用请求（函数名 + 参数）
4. 外部系统执行函数，返回结果
5. 模型根据结果生成最终回答

**典型用途：**
- 实时数据查询（天气、股价、数据库）
- 操作执行（发邮件、创建日历、写文件）
- 多步骤工作流编排

**与 MCP 的关系：** Function Calling 是单次调用机制，MCP 是标准化的工具连接协议，二者可结合使用。

## 7. Prompt Engineering（提示词工程）

**一句话定义：** 通过精心设计输入给模型的文本指令，最大化模型输出质量的技术和方法论。

**常用技巧：**

| 技巧 | 说明 | 示例 |
|------|------|------|
| **Zero-shot** | 直接描述任务，不提供示例 | "将以下文本翻译成英文：..." |
| **Few-shot** | 提供少量示例引导模型 | 给出2-3个"输入→输出"范例 |
| **Chain-of-Thought** | 要求模型逐步推理 | "请一步步思考..." |
| **Role Prompting** | 赋予模型特定角色 | "你是一位资深数据分析师..." |
| **XML 标签结构化** | 用标签分隔指令与内容 | `<context>...</context>` |

**进阶方向：** System Prompt 设计、Multi-turn 对话管理、Prompt 评估与优化

## 8. Vibe Coding（氛围编程）

**一句话定义：** 由 Andrej Karpathy 提出，描述一种完全依赖 AI 辅助、以"感觉对了就行"为核心的全新编程方式。

**核心理念：**
- 开发者用自然语言描述需求，AI 生成代码
- 不深入阅读每一行代码，而是测试运行结果
- 遇到 Bug 直接把报错粘给 AI，让它修复
- 代码质量让 AI 负责，人类专注于产品逻辑

**适用场景：** 原型验证、个人项目、非核心模块快速实现

**争议点：**
- ✅ 极大降低编程门槛，释放创意
- ⚠️ 对生产级系统存在安全和质量风险
- ⚠️ 开发者可能逐渐失去对代码的深度理解

**代表工具：** Cursor、Windsurf、Claude Code、GitHub Copilot
